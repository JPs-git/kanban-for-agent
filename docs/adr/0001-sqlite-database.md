# 0001 - 使用 SQLite 作为数据库

## 状态

已接受

## 上下文

Kanban Board 项目需要一个轻量级、易于部署的数据库解决方案。考虑的选项包括：

- SQLite：嵌入式数据库，无需独立服务器
- PostgreSQL：功能强大的关系型数据库
- MongoDB：NoSQL 文档数据库

### 决策因素

| 因素 | SQLite | PostgreSQL | MongoDB |
|------|--------|------------|---------|
| 部署复杂度 | 低（文件级） | 高（独立服务） | 中（独立服务） |
| 资源需求 | 极低 | 高 | 中 |
| 开发体验 | 简单（文件操作） | 需要服务器 | 需要服务器 |
| 数据完整性 | 支持事务 | 完整支持 | 有限 |
| 扩展性 | 有限 | 高 | 高 |

## 决策

选择 SQLite 作为数据库解决方案。

## 理由

1. **部署简单**：SQLite 是嵌入式数据库，无需安装和配置独立服务器，只需一个文件即可运行
2. **资源高效**：适合资源受限的环境和桌面应用场景
3. **开发便捷**：开发过程中无需启动数据库服务，简化开发流程
4. **事务支持**：SQLite 支持 ACID 事务，满足 kanban 应用的基本数据完整性需求
5. **文件格式**：数据库以单个文件存储，便于备份和迁移

## 影响

- 数据库迁移需要通过脚本手动执行
- 高并发场景下性能可能受限
- 不适合大规模分布式部署

## 替代方案

未来如果需要更高的扩展性，可以考虑迁移到 PostgreSQL 或其他数据库。
