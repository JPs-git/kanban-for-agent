# Kanban Board v0.3 UI设计文档

## 1. 设计概述

### 1.1 设计目标

根据v0.3版本的产品需求文档，本次UI设计旨在解决以下核心问题：
- 修复PC上看板不居中的问题
- 优化board宽度，确保能容纳4个column而不需要横向滚动
- 添加用户管理功能界面
- 完善错误处理界面设计

### 1.2 设计原则

- **简洁明了**：界面设计简洁，突出核心功能，减少视觉干扰
- **响应式设计**：适配不同屏幕尺寸，确保在PC、平板和移动设备上都能正常显示
- **一致性**：保持整体设计风格一致，包括颜色、字体、间距等
- **用户友好**：操作流程直观，反馈及时，降低用户学习成本
- **可访问性**：确保界面元素有足够的对比度，支持键盘导航

### 2.3 布局设计

#### 2.3.1 整体布局

- **顶部**：看板标题、添加卡片按钮和用户管理按钮
- **中部**：添加卡片表单（展开/收起）
- **底部**：状态列（待处理、进行中、已完成、已拒绝）
- **右侧**：用户管理面板（可展开/收起）

#### 2.3.2 响应式布局

- **PC端**：
  - 看板居中显示，最大宽度设置为1400px
  - 4个状态列并排显示，每个列宽度自适应
  - 用户管理面板作为侧边栏，可展开/收起

- **平板端**：
  - 看板居中显示，最大宽度设置为100%

  - 用户管理面板作为抽屉式菜单，从右侧滑出

- **移动端**：
  - 看板全屏显示
  - 状态列垂直堆叠显示
  - 用户管理面板作为全屏模态框显示

## 3. 交互设计

### 3.1 布局交互

#### 3.1.1 看板居中

- **PC端**：通过CSS的margin: 0 auto实现看板在页面中居中显示
- **响应式**：在不同屏幕尺寸下，看板宽度会自动调整，保持居中显示

#### 3.1.2 Board宽度优化

- **PC端**：设置board的最大宽度为1400px，确保4个column可以完全容纳
- **column宽度**：每个column的最小宽度设置为320px，通过flex布局实现自适应宽度
- **间距**：column之间的间距设置为20px，确保布局美观

### 3.2 用户管理功能交互

#### 3.2.1 用户管理按钮

- **位置**：位于看板头部右侧，添加卡片按钮旁边
- **样式**：次要按钮样式
- **点击效果**：点击后展开/收起用户管理面板

#### 3.2.2 用户管理面板

- **展开/收起**：点击用户管理按钮后，面板从右侧滑入/滑出
- **用户列表**：显示当前系统中的所有用户，包括用户ID和名称
- **搜索功能**：顶部有搜索框，可根据用户名称搜索
- **排序功能**：支持按用户ID或名称排序
- **添加用户**：底部有添加用户按钮，点击后显示添加用户表单
- **编辑用户**：每个用户项右侧有编辑按钮，点击后显示编辑用户表单
- **删除用户**：每个用户项右侧有删除按钮，点击后显示确认对话框

#### 3.2.3 用户表单

- **添加用户**：
  - 表单包含用户ID和名称输入框
  - 表单验证：用户ID和名称不能为空
  - 提交后，用户列表实时更新

- **编辑用户**：
  - 表单预填当前用户信息
  - 表单验证：用户ID和名称不能为空
  - 提交后，用户列表实时更新

- **删除用户**：
  - 确认对话框：提示用户确认删除操作
  - 确认后，用户列表实时更新
  - 取消后，返回用户列表

### 3.3 错误处理交互

#### 3.3.1 全局错误处理

- **错误提示**：使用非阻塞的消息提示，从顶部滑入，一段时间后自动消失
- **错误类型**：
  - 成功：绿色背景，显示成功信息
  - 警告：黄色背景，显示警告信息
  - 错误：红色背景，显示错误信息

#### 3.3.2 表单错误处理

- **实时验证**：输入过程中实时验证，显示错误提示
- **提交验证**：提交表单时验证，显示错误提示
- **错误提示位置**：错误提示显示在输入框下方

#### 3.3.3 API错误处理

- **加载状态**：API请求时显示加载指示器
- **错误状态**：API请求失败时显示错误提示
- **重试机制**：错误提示中包含重试按钮

### 3.4 卡片管理交互

#### 3.4.1 添加卡片

- **点击添加卡片按钮**：展开添加卡片表单
- **表单验证**：标题不能为空，内容可以为空
- **提交**：点击提交按钮后，卡片添加到对应状态列
- **取消**：点击取消按钮后，收起添加卡片表单

#### 3.4.2 编辑卡片

- **点击卡片编辑按钮**：打开编辑模态框
- **表单预填**：模态框中预填当前卡片信息
- **保存**：点击保存按钮后，卡片信息更新
- **取消**：点击取消按钮后，关闭模态框

#### 3.4.3 删除卡片

- **点击卡片删除按钮**：显示确认对话框
- **确认**：点击确认按钮后，卡片被删除
- **取消**：点击取消按钮后，返回卡片列表

#### 3.4.4 拖拽功能

- **拖拽开始**：卡片变为半透明，显示拖拽提示
- **拖拽过程**：鼠标悬停在状态列上时，状态列背景变为浅蓝色
- **拖拽结束**：卡片移动到新的状态列，状态更新

## 4. 用户体验说明

### 4.1 布局优化体验

- **PC端**：看板居中显示，4个状态列并排展示，无需横向滚动，布局美观大方
- **平板端**：响应式布局，确保在较小屏幕上也能正常显示
- **移动端**：状态列垂直堆叠，操作简单直观

### 4.2 用户管理体验

- **便捷性**：用户管理功能集成在看板界面中，无需切换页面
- **实时性**：用户增删改操作后，用户列表实时更新，确保数据一致性
- **安全性**：删除用户时需要确认，防止误操作
- **易用性**：表单设计简洁明了，验证及时，反馈明确

### 4.3 错误处理体验

- **友好性**：错误提示采用非阻塞式设计，不影响用户操作
- **及时性**：错误信息及时显示，帮助用户快速定位问题
- **明确性**：错误信息清晰明了，指导用户如何解决问题
- **一致性**：错误处理机制统一，提升用户体验

### 4.4 卡片管理体验

- **流畅性**：拖拽功能流畅，状态更新及时
- **灵活性**：支持多种方式管理卡片，包括拖拽、编辑和删除
- **直观性**：卡片状态和分配人员清晰可见
- **高效性**：添加和编辑卡片操作简单快捷

## 5. 实现方案

### 5.1 布局优化实现

- **CSS修改**：
  - 调整.kanban-board的max-width为1400px
  - 调整.status-column的min-width为320px
  - 确保board-content使用flex布局，且overflow-x为auto

- **响应式设计**：
  - 使用媒体查询，在不同屏幕尺寸下调整布局
  - 在移动端将board-content改为flex-direction: column

### 5.2 用户管理功能实现

- **组件设计**：
  - 创建UserManagement组件，包含用户列表、搜索、排序和操作功能
  - 创建UserForm组件，用于添加和编辑用户
  - 创建UserDeleteConfirm组件，用于确认删除操作

- **状态管理**：
  - 使用React状态管理用户列表
  - 实现用户增删改的API调用
  - 实时更新用户列表

- **交互实现**：
  - 使用CSS transitions实现面板的展开/收起动画
  - 使用React hooks管理表单状态和验证
  - 实现用户搜索和排序功能

### 5.3 错误处理实现

- **全局错误处理**：
  - 创建ErrorBoundary组件，捕获全局错误
  - 创建Toast组件，显示非阻塞的错误提示

- **API错误处理**：
  - 在useKanban hook中添加错误处理逻辑
  - 实现加载状态和错误状态的UI展示

- **表单错误处理**：
  - 在表单组件中添加验证逻辑
  - 实现实时验证和错误提示

## 7. 结论
