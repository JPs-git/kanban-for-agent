# Kanban Board v0.3 产品需求文档

## 1. 产品概述

Kanban Board 是一个轻量级的任务管理工具，用于可视化工作流程和任务状态。v0.3 版本基于 v0.2 版本的用户反馈进行修复和优化，提升用户体验和功能完整性。

## 2. 核心功能

### 2.1 功能模块

| 模块名称 | 功能描述 | 优先级 |
|---------|---------|--------|
| 布局优化 | 修复PC上看板不居中问题，确保board足够大放得下4个column | 高 |
| 用户管理 | 添加手动增删改用户的功能 | 高 |
| 错误处理 | 修复添加card时不输入content前端报错的问题 | 高 |
| 基础功能 | 保持v0.2版本的核心功能 | 中 |

### 2.2 功能详情

#### 2.2.1 布局优化

- **看板居中**：
  - 确保PC上看板在页面中居中显示
  - 优化响应式布局，确保在不同屏幕尺寸下都能正常显示

- **Board宽度优化**：
  - 确保board足够大，能够容纳4个column而不需要横向滚动
  - 调整column宽度和间距，确保布局美观合理

#### 2.2.2 用户管理

- **用户列表**：
  - 显示当前系统中的所有用户
  - 支持用户搜索和排序

- **添加用户**：
  - 支持添加新用户，包括用户ID和名称
  - 表单验证，确保用户信息完整

- **编辑用户**：
  - 支持修改现有用户的信息
  - 实时更新用户列表

- **删除用户**：
  - 支持删除不需要的用户
  - 删除前确认，防止误操作

- **用户分配**：
  - 在卡片创建和编辑时，下拉菜单中显示最新的用户列表
  - 支持取消分配（未分配状态）

#### 2.2.3 错误处理

- **后端错误处理**：
  - 确保后端API允许content字段为空
  - 完善错误处理机制，返回清晰的错误信息

- **前端错误处理**：
  - 添加全局错误处理机制，防止页面崩溃
  - 使用非阻塞的message提示，提升用户体验
  - 对API请求添加loading状态和错误提示

#### 2.2.4 基础功能

- **卡片管理**：
  - 创建、编辑、删除卡片
  - 支持输入卡片标题和内容，内容可以为空
  - 支持选择初始状态和分配人员

- **状态管理**：
  - 卡片状态的拖拽和手动修改
  - 状态包括：待处理、进行中、已完成、已拒绝

- **人员分配**：
  - 卡片分配给团队成员
  - 人员名称在下拉菜单中清晰可见

## 3. 技术实现

### 3.1 前端实现

- **框架**：React + TypeScript
- **拖拽功能**：react-dnd
- **样式**：CSS
- **编辑模式**：模态框组件
- **错误处理**：添加全局错误处理和消息提示

### 3.2 后端实现

- **框架**：Node.js + Express
- **数据库**：MongoDB
- **API**：RESTful API
- **用户管理**：添加用户相关的API端点

## 4. 修复内容

| 问题编号 | 问题描述 | 修复方案 |
|---------|---------|----------|
| 1 | PC上看板不居中 | 优化CSS布局，确保看板在页面中居中显示 |
| 2 | board横向有滚动条，应该足够大放得下4个column | 调整board宽度和column宽度，确保4个column可以完全容纳 |
| 3 | 应添加手动增删改用户的功能 | 实现用户管理界面和API，支持增删改用户 |
| 4 | 添加card时不输入content前端报错，页面消失 | 修复后端API允许content为空，前端添加错误处理机制 |

## 5. 用户界面设计

### 5.1 整体布局

- 顶部：看板标题、添加卡片按钮和用户管理按钮
- 中部：添加卡片表单（展开/收起）
- 底部：状态列（待处理、进行中、已完成、已拒绝）
- 右侧：用户管理面板（可展开/收起）

### 5.2 卡片设计

- 卡片背景：白色
- 卡片边框：浅灰色
- 卡片阴影：轻微阴影效果
- 卡片内容：标题、内容、分配人员、状态

### 5.3 编辑模态框

- 模态框背景：半透明黑色遮罩
- 模态框内容：白色背景
- 模态框大小：固定宽度，自适应高度
- 模态框内容：标题输入框、内容文本域、状态选择下拉框、人员分配下拉框、保存和取消按钮

### 5.4 用户管理面板

- 面板背景：白色
- 面板边框：浅灰色
- 面板内容：用户列表、添加用户按钮、编辑和删除用户操作
- 表单设计：简洁明了，包含必要的输入字段和验证

## 6. 验收标准

| 测试项 | 验收标准 |
|-------|----------|
| 看板居中 | PC上看板在页面中居中显示，布局美观 |
| Board宽度 | board能够容纳4个column而不需要横向滚动 |
| 用户管理 | 可以手动增删改用户，用户列表实时更新 |
| 错误处理 | 添加card时不输入content不会导致前端报错和页面消失 |
| 卡片管理 | 可以创建、编辑、删除卡片，内容可以为空 |
| 状态管理 | 卡片可以在不同状态列之间拖拽，状态更新正确 |
| 人员分配 | 卡片可以分配给用户，下拉菜单中显示最新的用户列表 |
| 响应式设计 | 在不同屏幕尺寸下都能正常显示 |

## 7. 版本计划

- **v0.3 版本**：修复上述问题，添加用户管理功能，提升系统稳定性和用户体验
- **v0.4 版本**：添加更多高级功能，如标签、截止日期、搜索等

## 8. 风险评估

| 风险项 | 影响程度 | 应对措施 |
|-------|----------|----------|
| 布局兼容性 | 低 | 测试不同浏览器和屏幕尺寸的显示效果 |
| 用户管理功能复杂度 | 中 | 采用模块化设计，确保代码可维护性 |
| 错误处理机制 | 中 | 完善前端和后端的错误处理，确保系统稳定性 |
| 数据一致性 | 低 | 确保用户数据和卡片数据的一致性 |
