# Kanban系统v0.2版本用户反馈文档

## 反馈时间
2026-03-11

## 反馈问题

### 1. 布局问题
- **问题描述**：PC上看板不居中
- **影响范围**：用户界面体验
- **优先级**：中

### 2. 布局问题
- **问题描述**：board横向有滚动条，应该足够大放得下4个column
- **影响范围**：用户界面体验
- **优先级**：中

### 3. 功能需求
- **问题描述**：应添加手动增删改用户的功能，便于用户操作可分配的人员
- **影响范围**：用户管理功能
- **优先级**：高

### 4. BUG反馈
- **问题描述**：添加card时不输入card content前端报错，页面消失
- **错误信息**：
  ```
  api.ts:14 POST http://localhost:3000/api/cards 500 (Internal Server Error)
  createCard @ api.ts:14
  addCard @ useKanban.ts:29
  handleAddCard @ KanbanBoard.tsx:77
  ```
- **预期结果**：可以添加不包含content的卡片
- **建议**：如果后端报错前端页面不应该崩溃，建议使用非阻塞的message提示
- **影响范围**：核心功能
- **优先级**：高

## 技术分析

### 问题4分析
- **错误原因**：后端API在处理空content时可能存在验证问题
- **前端问题**：前端没有正确处理后端错误，导致页面崩溃
- **建议解决方案**：
  1. 后端修改：确保content字段可以为空
  2. 前端修改：添加错误处理机制，使用非阻塞的message提示

## 修复建议

1. **布局问题修复**：
   - 调整看板布局，使其在PC上居中显示
   - 优化board宽度，确保能容纳4个column而不需要横向滚动

2. **功能添加**：
   - 实现用户管理功能，包括增删改用户
   - 确保用户管理界面直观易用

3. **BUG修复**：
   - 修复后端API，允许content字段为空
   - 改进前端错误处理，使用message提示而不是页面崩溃

## 结论

这些问题需要在v0.3版本中优先解决，特别是BUG问题和用户管理功能，以提升系统的稳定性和用户体验。
