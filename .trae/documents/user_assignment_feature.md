# Kanban系统用户分配功能需求文档

## 1. 产品概述

本需求文档描述了为Kanban系统的卡片（Card）添加用户分配功能的详细要求。通过此功能，用户可以在创建卡片时指定分配人员，或在卡片创建后修改分配人员，实现任务的明确归属和责任追踪。

## 2. 功能需求

### 2.1 核心功能

| 功能点 | 描述 |
|-------|------|
| 用户分配字段 | 在卡片中添加分配人员字段，支持选择用户或设置为"未分配" |
| 创建卡片时分配用户 | 在创建卡片时可以指定分配人员 |
| 修改卡片分配用户 | 在编辑卡片时可以修改分配人员 |
| 显示分配状态 | 在卡片上显示当前分配状态和人员 |
| 用户选择界面 | 提供用户选择下拉列表，包含所有可用用户和"未分配"选项 |

### 2.2 数据结构变更

#### 2.2.1 后端模型变更

**Card模型**
| 字段名 | 类型 | 描述 | 约束 |
|-------|------|------|------|
| assignee | String | 分配人员ID | 可选，默认为null |
| assigneeName | String | 分配人员姓名 | 可选，默认为"未分配" |

#### 2.2.2 前端类型变更

**Card接口**
| 字段名 | 类型 | 描述 |
|-------|------|------|
| assignee | string | 分配人员ID |
| assigneeName | string | 分配人员姓名 |

**CardCreate接口**
| 字段名 | 类型 | 描述 |
|-------|------|------|
| assignee | string | 分配人员ID |
| assigneeName | string | 分配人员姓名 |

**CardUpdate接口**
| 字段名 | 类型 | 描述 |
|-------|------|------|
| assignee | string | 分配人员ID |
| assigneeName | string | 分配人员姓名 |

## 3. 非功能需求

| 需求点 | 描述 | 验收标准 |
|-------|------|---------|
| 性能要求 | 用户分配操作响应时间不超过1秒 | 在网络正常情况下，分配人员操作响应时间≤1秒 |
| 可用性 | 系统应保持99.9%的可用性 | 系统运行稳定，无因用户分配功能导致的崩溃 |
| 兼容性 | 支持主流浏览器（Chrome、Firefox、Safari、Edge） | 在指定浏览器中功能正常显示和操作 |
| 安全性 | 用户只能分配系统中存在的用户 | 系统应验证分配的用户是否存在 |

## 4. 用户故事

| 用户故事 | 场景 | 目标 |
|---------|------|------|
| 作为项目管理者，我希望在创建卡片时能够指定分配人员，以便明确任务责任 | 创建新任务卡片时，需要选择具体的负责人员 | 确保每个任务都有明确的负责人 |
| 作为团队成员，我希望看到卡片上显示的分配人员，以便了解任务归属 | 查看卡片时，能够清晰看到当前分配的人员 | 快速识别任务的负责人 |
| 作为项目管理者，我希望能够修改卡片的分配人员，以便根据工作负荷和技能调整任务分配 | 当工作负载变化或任务需求变更时，需要重新分配任务 | 灵活调整任务分配，确保工作高效进行 |
| 作为团队成员，我希望能够将卡片设置为"未分配"，以便表示任务暂时不需要特定人员负责 | 当任务处于待分配状态时，需要将其标记为未分配 | 清晰标识未分配的任务，便于后续分配 |

## 5. 验收标准

| 测试项 | 验收标准 |
|-------|---------|

2. 下拉列表包含所有可用用户和"未分配"选项 |

## 6. 实现考虑

### 6.1 后端实现

1. 修改Card模型，添加assignee和assigneeName字段
2. 更新卡片创建和更新API，支持处理用户分配信息
3. 实现用户验证逻辑，确保分配的用户存在

### 6.2 前端实现

1. 修改Card组件，添加分配人员显示和编辑功能
2. 更新类型定义，添加assignee相关字段
3. 实现用户选择下拉组件
4. 更新卡片创建和编辑表单，添加用户分配选项

### 6.3 数据迁移

对于现有卡片，需要将assignee字段设置为null，assigneeName设置为"未分配"，以保持数据一致性。

## 7. 风险评估

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 用户系统未实现 | 无法进行用户分配 | 需先实现用户系统或使用模拟数据 |
| 数据迁移问题 | 现有卡片可能出现显示异常 | 制定详细的数据迁移计划，确保平滑过渡 |
| 性能影响 | 大量用户选择可能影响性能 | 实现用户搜索和分页功能，优化选择体验 |
