# 0006 - 前端路由与布局架构

## 状态

已接受

## 上下文

Kanban Board 前端应用需要从单页面应用扩展为多页面应用，支持：
- 首页：显示看板组件
- 关于页：显示版本信息和环境信息
- 未来可能添加管理或统计页面

需要设计路由方案和布局架构。

### 路由方案选项

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| react-router-dom | React 社区最流行的路由库 | 功能完整、社区成熟、支持嵌套路由 | 体积较大 |
| wouter | 轻量级路由库 | 体积小、API简洁 | 功能有限 |
| 自定义状态管理 | 使用 useState/useContext | 零依赖 | 需要自己实现路由逻辑 |

### 布局方案选项

| 方案 | 描述 | 适用场景 |
|------|------|----------|
| 全局 Header + 内容区域 | 顶部固定导航 + 可滚动内容区 | 标准企业应用 |
| 侧边栏导航 | 左侧导航菜单 | 功能较多的复杂应用 |
| 标签页式 | 类似浏览器标签页 | 需要频繁切换的场景 |

## 决策

1. **选择 react-router-dom 作为路由库**
2. **采用全局 Header + 内容区域的布局方案**
3. **创建集中式路由配置**
4. **抽象统一的布局组件**

## 理由

### react-router-dom

1. **社区成熟**：React 社区最流行的路由解决方案，文档和资源丰富
2. **功能完整**：支持嵌套路由、路由参数、导航守卫等高级功能
3. **类型安全**：良好的 TypeScript 支持
4. **生态完善**：与 React 生态深度集成

### 全局 Header 布局

1. **简洁直观**：用户熟悉的顶部导航模式
2. **节省空间**：相比侧边栏，更适合功能相对集中的应用
3. **响应式友好**：移动端可以轻松折叠为汉堡菜单

### 集中式路由配置

1. **易于维护**：所有路由定义集中在一个文件
2. **可扩展性强**：新增页面只需添加配置
3. **类型安全**：使用 RouteObject 类型确保配置正确

### 统一布局组件

1. **代码复用**：避免重复的布局逻辑
2. **一致性**：确保各页面布局统一
3. **可维护性**：布局修改只需在一处进行

## 影响

### 文件结构变化

```
src/
├── components/
│   ├── Layout.tsx          # 全局布局容器
│   ├── DashboardLayout.tsx # 仪表板布局
│   └── Header.tsx          # 顶部导航
├── pages/
│   ├── Home.tsx            # 首页
│   ├── About.tsx           # 关于页
│   └── NotFound.tsx        # 404 页面
├── routes/
│   └── index.tsx           # 路由配置
└── App.tsx                 # 应用入口
```

### 路由配置

| 路径 | 页面 | 描述 |
|------|------|------|
| `/` | Home | 首页，显示看板组件 |
| `/about` | About | 关于页，显示版本和环境信息 |
| `*` | NotFound | 404 页面 |

### 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│ Layout (h-screen)                                          │
│   ├── Header (h-16)                                       │
│   │   Kanban Board                    Home    About         │
│   └── main (flex-1)                                       │
│         └── DashboardLayout                                │
│               ├── actions (操作按钮区)                      │
│               └── content (页面内容)                       │
└─────────────────────────────────────────────────────────────┘
```

## 替代方案

- **wouter**：如果项目对包体积有严格要求，可以考虑
- **侧边栏布局**：如果未来功能大幅增加，可以考虑迁移
- **分散式路由配置**：小型项目可以直接在 App.tsx 中定义
