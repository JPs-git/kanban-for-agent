# 0002 - 使用 Express.js 作为后端框架

## 状态

已接受

## 上下文

Kanban Board 项目需要一个轻量级的后端框架来提供 REST API。考虑的选项包括：

- Express.js：Node.js 生态中最流行的 Web 框架
- NestJS：企业级框架，支持 TypeScript
- Fastify：高性能替代方案

### 决策因素

| 因素 | Express.js | NestJS | Fastify |
|------|------------|--------|---------|
| 学习曲线 | 低 | 高 | 中 |
| 社区支持 | 非常丰富 | 良好 | 中等 |
| 性能 | 良好 | 良好 | 优秀 |
| TypeScript支持 | 良好 | 原生支持 | 良好 |
| 灵活性 | 高 | 中等 | 高 |

## 决策

选择 Express.js 作为后端框架。

## 理由

1. **成熟稳定**：Express.js 是 Node.js 生态中最成熟的 Web 框架，经过多年生产验证
2. **学习成本低**：API 简洁直观，易于上手
3. **社区资源丰富**：大量中间件和教程资源可用
4. **灵活性高**：允许自由组织代码结构，适合小型项目
5. **TypeScript 兼容**：虽然不是原生支持，但通过类型定义可以很好地工作

## 影响

- 需要手动配置 TypeScript 支持
- 缺乏内置的依赖注入系统
- 需要自己组织项目结构

## 替代方案

未来如果项目复杂度增加，可以考虑迁移到 NestJS。
